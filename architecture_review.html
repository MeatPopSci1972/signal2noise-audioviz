GoF Pattern Opportunities

STRATEGY Priority 1
synth() is a 200-line switch over 16 types. vfxTick() is a massive if/else over 10 object kinds. PAL_FIRE is hardcoded.
SynthStrategy map: { kick: { play(when,vol) } }. VFXStrategy map: { ring: { update(o), render(o,vx) } }. FirePaletteStrategy: { map(v):[r,g,b] }. New type = one map entry, zero edits to existing code.
OBSERVER / EventBus Priority 2
trigViz() has 7 direct responsibilities. Scheduler knows about fire, VFX, bar chart, creatures. New effect = edit trigViz().
bus.emit('note', {ci, delay, vol}). FireEngine, VFXPool, BarChart, Creatures all subscribe. trigViz() is deleted. New effects subscribe without touching existing code.
TEMPLATE METHOD Priority 3
VFX objects share init→update→render→expire lifecycle but it's duplicated across 10 if/else branches. Balloons and creatures run in separate loops.
VFXObject base: { update(), render(vx), isExpired() }. Subclasses override steps. vfxTick() becomes a single forEach. Three loops unified.
COMMAND Priority 4
loadPreset() has 3 magic string branches (__random, __clear, named). No undo. Preset actions are not first-class objects.
PresetCommand: { execute(), undo() }. CommandInvoker keeps history stack. RandomCommand, ClearCommand, LoadCommand implement the interface. Undo/redo for free.
FACADE Priority 5
Scheduler has 8 direct global dependencies. buildGrid() has 4. Nothing can be tested in isolation.
AudioEngine facade: { play(ci,t), vol(ci) }. SequencerState facade: { getStep(ci,s), numSteps }. VFXConfig facade: { getEffects(ci) }. Injectable in tests.
STATE MACHINE Priority 6
Sequencer STOPPED/PLAYING managed by scattered booleans. Sampler UNARMED/ARMED/RECORDING/PROCESSING managed by isRecording + null-checks.
SequencerFSM + SamplerFSM with explicit transitions. Impossible transitions caught at the state boundary. Each state owns its enter/exit side effects.
OBJECT POOL Priority 7
VFX objects allocated as plain literals on every note hit, GC'd when expired. High allocation rate at intensity=3 with all effects active.
Pre-allocate 300 objects at startup. acquire(kind) resets and returns a pooled object. release(obj) returns it. vfxObjs/balloons/creatures unified under one PoolManager.
BUILDER Priority 8
Preset objects are plain { bpm, swing, pattern } literals. analyzeAndBuildPreset() mixes analysis with construction across 40 lines with no validation.
PresetBuilder: .setBpm(n).setSwing(n).setPattern(p).build(). Validates bpm range, pattern dimensions at build() time. Separates analysis from construction.

Refactor Roadmap

1STRATEGYKills synth() switch + vfxTick() if/else. New drum/VFX type = one new map entry, zero edits to existing code. Biggest single unlock.effort: med
2OBSERVER / EventBusDeletes trigViz() entirely. Scheduler stops knowing about visuals. Each subsystem independently extensible.effort: med
3TEMPLATE METHODUnifies 3 separate update loops (vfxObjs, balloons, creatures) into one. Adds clean extension point for future VFX types.effort: med
4COMMANDloadPreset() magic strings become typed Command objects. Undo/redo history comes for free.effort: med
5FACADEPrerequisite for unit testing. Each subsystem gets a narrow injectable interface. Unlocks the 22 missing tests.effort: high
6STATE MACHINEEliminates scattered boolean guards. Impossible transitions become explicit errors. Sampler especially benefits.effort: med
7OBJECT POOLReduces GC jank at high intensity. Meaningful only at vizIntensity=3 with all VFX active — defer until felt.effort: med-hi
8BUILDERCleaner preset construction with validation at build() time. Low effort, nice-to-have.effort: low
